# R1.3 非破坏发布差异与一致性检查报告

## 实现

- `release_diff.py` 不再 checkout 基线和 master，改为通过 `git show <baseline>:packages.toml` 读取历史索引，并通过 GitPython 读取提交范围。
- 默认输出单一 JSON，包含 baseline/head、包 create/update/delete 和提交摘要；可显式写 Markdown 报告，失败也稳定输出 JSON 错误并返回非零。
- 新增 `release_check.py`，统一检查 `version_config.py`、运行时头文件、CMSIS Pack、core/PikaStdLib 包版本、各包最新 commit 可解析性、可选 tag/HEAD 和工作区清洁状态。
- 不要求 clean 时脏工作区作为 warning；正式发布使用 `--require-clean` 升级为错误。传入 tag 时可使用 `--require-tag-at-head` 检查不可移动发布点。
- `PackageReleaseList.from_text()` 支持从 Git 对象内容解析索引，无需临时 checkout 或临时工作树。

## 验证

- 发布工具单元测试合计 6/6 通过，覆盖三类版本源解析、历史 TOML 文本解析、Markdown 渲染、SemVer patch 和格式保持。
- 所有发布脚本及测试 `py_compile` 通过。
- 在真实仓库运行 `release_diff.py v1.13.4`：返回 89 个提交、1 个索引版本变化；运行前后 `git status --porcelain` 完全一致。
- 在当前元数据运行 `release_check.py --expected-version 1.13.4`：明确以非零退出，唯一版本错误为 CMSIS `1.13.0 != 1.13.4`，当前任务修改作为 dirty warning 可见。
- 首次 Docker heredoc 集成探针因未传 `-i` 无输出，不计为验证；补 `docker run -i` 后获得上述有效 JSON 结果。
- `git diff --check` 通过。
