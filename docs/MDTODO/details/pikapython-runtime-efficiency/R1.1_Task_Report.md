# R1.1 基线报告

- 基线构造：
  - 在 NC01 的 `/tmp/pikapython-runtime-efficiency/baseline-src` 构造与最终源码语义一致但撤销本轮性能补丁的副本。
  - 两边使用 GCC 14.2.0、CMake 3.31.6、Release、默认配置和同一生成器。
  - LVGL 因 Gitee 子模块不可匿名拉取而隔离；hashlib 因 mbedTLS 3 API 不兼容而隔离。
- 默认配置基线：
  - core archive：`text=1072241`、`data=125784`、`bss=24338`。
  - 测试二进制：`text=3693084`、`data=202656`、`bss=82784`。
  - `function_call_1000` CPU 中位数：`8.1646 ms`。
- 调用资源模型：
  - 基线每次参数绑定申请三块 scratch，分别为 argv 和两个类型列表缓冲。
  - 同时存活的请求字节总量为 64 位 Linux 640 字节、32 位目标 576 字节。
  - 本轮以 allocator 调用数、请求总字节和最终 `data/bss` 作为 RAM 门槛；未引入常驻字段。
