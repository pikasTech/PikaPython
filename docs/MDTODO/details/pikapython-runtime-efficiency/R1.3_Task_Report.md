# R1.3 函数调用优化报告

- 修改：
  - 参数是否包含 `*args` / `**kwargs` 的检查改为从 VM 栈顶一次反向扫描，复杂度由 O(n²) 降为 O(n)。
  - argv 与两个类型列表缓冲合并为一次连续 scratch 申请，allocator 调用由每次 3 次降为 1 次，请求总字节不变。
  - 关键字字典从每插入一项 reverse 改为收集完成后 reverse 一次，复杂度由 O(k²) 降为 O(k)。
- RAM 边界：
  - 未增加结构体字段、常驻缓存或大栈数组。
  - 32 位目标 scratch 请求仍为 576 字节；64 位 Linux 仍为 640 字节。
- 测试：
  - 新增多关键字顺序测试。
  - 默认参数、`*args`、关键字绑定、dict items 及五配置相关回归通过。
