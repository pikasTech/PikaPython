# R4.2 修复报告

## Parser 和字节码

- 严格校验 `try:` 和 `except` 子句。
- 保存 simple reference 异常类型和可选 alias。
- 每个新格式 `EXP` 后生成 `JEZ 1`，只执行首个匹配 handler。
- bare handler 之后禁止继续声明 typed handler。
- handler 链结束时生成 `GER`、`RIS`，传播未匹配异常。

## Runtime

- 在 `PikaVMError` 的对齐空隙保存异常类别，常驻 RAM 不增加。
- 从现有错误输出前缀、显式异常 constructor 和异常实例提取类别。
- 使用紧凑 time33 hash 表和父类表实现 Python 3 异常层级匹配。
- `raise ValueError()` 等内建异常实例不再被静默忽略。
- 每个新源码行开始时清除上一行的异常类别，避免错误恢复后类型状态污染。
- 旧格式 `EXP` 没有后续 `JEZ` 时继续使用原无条件捕获行为，兼容已有
  `.py.o` 和固化字节码。

## 解析测试

- 新增 6 个非法场景：非法 try、缺失类型、缺失 alias、tuple exception、
  bare 后继续 handler、重复 `as`。
- 新增 typed、多 handler、alias 和 bare fallback 合法邻接测试。
- 新增固定旧格式 ASM 回归，验证无后续 `JEZ` 的 `EXP` 仍按旧语义捕获。
