# R4.3 探测与回归报告

## 场景矩阵

- 正式运行时矩阵包含 23 个独立 root 场景。
- 临时探针额外覆盖 3 个场景，总计 26 个。
- 覆盖 bare、类型命中和错配、多 handler、bare fallback、alias、异常
  constructor、异常实例、异常父类、嵌套错配传播、非法 raise、return、
  break、continue 和错误后恢复。

## 结果

- 正式异常测试、parser typed except 测试和原有 except 测试全部通过。
- 每个正式运行场景析构后 `pikaMemNow()` 为 0。
- 固定旧格式 ASM 兼容回归通过，析构后 `pikaMemNow()` 为 0。
- 核心组合回归执行 197 项，通过 196 项。
- `PIKA_SYNTAX_LEVEL=0` minimal 核心库完整编译通过。
- 唯一失败 `string.str_issue1` 使用当前 parser 不支持的复合 `and` 条件，
  在隔离包中为既有失败，与异常改动无关。

## 剩余边界

- handler 内再次抛出到外层仍受线程级单 try 状态限制。
- 隔离包删除了 LVGL、crypto 和部分测试模块，网络类测试未运行。
- 重新绑定后的 unittest 模块在 HEAD VM/Obj A/B 中均出现相同既有失败，
  不作为本轮回归判定。
