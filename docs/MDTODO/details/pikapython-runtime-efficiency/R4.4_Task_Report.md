使用同一 Release 配置、同一隔离 core 源集和 `-O3 -DNDEBUG`。跨 worktree 通过 `-ffile-prefix-map=<worktree>=/src` 归一化 `__FILE__`，避免绝对路径长度污染只读数据。

| 配置 | 指标 | 修复前基线 | 正确性修复后 | 最终 | 最终净变化 |

- 正确性修复在 default 原始增加 `2480 B text`，minimal 通过异常编译期裁剪保持零增量。
- 补偿项对 parser 的两个 Flash 大函数和异常类型解析、`EXP`、`RIS` 冷路径使用 GCC 定向 `Os`；不改变 VM 热路径、全局优化等级和其他编译器行为。
- parser 尺寸优化可能降低一次性解析速度；按项目原则未做 parser 性能优化或吞吐声明。runtime/VM 热路径没有引入性能改动，本任务不宣称 runtime 性能收益。
- 最终 default/minimal 的 text、data、bss 均不高于修复前基线。

## 回归

- default 异常定向：7/7 通过，覆盖 typed handler、alias、旧格式无类型 `EXP`、合法/非法 parser 邻接和原有 except。
- default 核心宽回归：304 项中 303 项通过；唯一失败 `datastruct.circlequeue` 含当前子集不支持的复合 `and` 条件，在同一隔离树替换修复前基线 parser 后稳定复现，因此不归因于本轮。排除该既有失败后为 303/303。
- minimal 完整测试目标编译成功；对象、栈、队列承诺范围 30/30 通过。
- default 与 minimal 构建结束后均恢复 `pika_config.h` 为 default canonical 内容。

## 隔离与坑

- LVGL 子模块缺 `lvgl.h`；系统 mbedTLS 与仓库旧 crypto API 不兼容；网络、crypto、LVGL 和缺实现的 `cjson_test` 按任务授权隔离。
- 生成器会扫描目录内全部 `.pyi`，只修改 `module_list.txt` 或 CMake 不能完整隔离模块；临时树需移走对应 `.pyi`、重新生成 binding，并排除直接依赖的测试源。
- `pull-core.sh` 必须从 `port/linux` 执行，生成器必须从 `port/linux/package/pikascript` 执行。
- 上述通用规则和跨 worktree 路径归一化要求已沉淀到 `pikapython` skill。

## 剩余边界

- handler 内再次抛出到外层仍受线程级单 try 状态限制。
- alias 在父类捕获子类时不能完整保留原始子类实例语义。
- 这两项继续由 #368 跟踪，不以局部补丁改变 VM/对象模型/字节码架构。
