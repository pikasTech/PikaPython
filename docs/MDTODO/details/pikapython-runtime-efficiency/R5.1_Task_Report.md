# R5.1 语法与 Python 3 兼容性探测报告

## 探测

在独立 Linux 源码树中批量探测 22 个解析期与运行期邻接场景，覆盖表达式、控制流、函数参数、异常、迭代和切片。参数缺失、未知关键字、重复位置/关键字、多余位置参数、不可调用对象及错误后恢复均已有稳定错误路径。

确认 stepped slice 存在兼容性缺口：`values[0:2:1]` 与 `values[::-1]` 会被 parser 接受，但 VM 的 `SLC` 只支持现有二元切片，前者返回错误结果，后者可能错误执行。

## 修复

- 不扩展 VM 或字节码，parser 在已有子集边界稳定拒绝两个及以上冒号的 stepped slice。
- 保留 `values[0:2]` 合法邻接测试，防止扩大拒绝范围。
- parser 单元测试覆盖正步长、负步长和过多字段三类非法输入。

## 残余风险

`except ValueError as err` 和 typed `except` 的条件/alias 存在既有语义债务，当前 parser 可能忽略条件或让首个分支捕获非匹配异常。修复需要梳理异常分派合同，超出本批局部边界，因此只记录，不仓促改变异常架构；后续由 [#368](https://github.com/pikasTech/PikaPython/issues/368) 跟踪。
