# R5.2 异常可见性与 runtime 状态报告

## 问题

`for item in 1:` 进入 `builtins_iter()` 后，`_arg_to_obj()` 返回空指针，原实现使用 `pika_assert` 终止进程；Linux 表现为 `Aborted`，嵌入式目标等价于死机，无法由 Python 异常处理。

## 修复

- 非可迭代对象不再触发断言。
- 设置 `TypeError: object is not iterable` 和 `PIKA_RES_ERR_INVALID_PARAM`，沿既有 runtime 错误通道返回。
- 新增回归测试验证 `except TypeError` 可捕获、后续合法语句继续执行、对象释放后 `pikaMemNow() == 0`。

名称错误、除零、不可调用对象和既有参数错误探测未发现新的崩溃或状态残留。
