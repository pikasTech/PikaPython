# R6.1 runtime/VM 热点基线

## 负载

- 使用仓库 `port/linux/performance` 的 `-pg`/gprof 入口，将临时 workload 改为预编译字节码后执行 200 万次 `add(i)` 默认参数调用。
- workload 无循环输出，校验最终结果为 `2000001`；parser 位于计时区外。
- 四次长负载耗时为 `5.096735s`、`4.834529s`、`4.913023s`、`4.493462s`，分配次数均为 `12009057`，缓存命中均为 `6007902`。

## 热点

- `__pikaVM_runByteCodeFrameWithState` self `19.75%`。
- `_obj_getObjWithKeepDeepth` self `10.89%`，为最大可独立局部优化热点。
- `obj_setArg_noCopy` self `5.06%`。
- `PikaVMFrame_loadArgsFromMethodArg` 与 `VM_instruction_handler_OPT` self 各 `4.05%`。

## 候选判定

普通函数名在 RUN 中调用 `obj_getHostObjWithIsTemp()`。当 `keepDeepth=1` 且路径无点号时，现实现仍复制路径并统计 token，随后循环零次直接返回当前对象。可在不改变返回值、临时对象状态和路径语义的情况下提前返回；该候选直接对应 10.89% hotspot，不涉及 parser、对象结构或常驻缓存。

Google Benchmark 五次重复显示 CPU CV 约 4%-17%，当前主机负载较高；短基准只用于后续交替 A/B 和控制组，不单独作为收益结论。
