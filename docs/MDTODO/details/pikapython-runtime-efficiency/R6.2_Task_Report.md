# R6.2 runtime 热路径局部优化

## 实现

- gprof 基线中 `_obj_getObjWithKeepDeepth` self 占比为 `10.89%`。
- 普通函数名通过 `obj_getHostObjWithIsTemp(..., keepDeepth=1)` 查询时，无点号路径原实现仍复制路径、统计 token，随后循环零次返回当前对象。
- 在路径长度断言之后增加无点号快速返回，保持超长路径断言、嵌套路径、临时对象状态和对象模型不变。
- 快速路径仅在 `!PIKA_NANO_ENABLE` 启用；minimal/nano 保持原代码，避免其 Flash 增量。

## 测试

- 新增 `object_test.get_host_obj_simple_and_nested_path`，覆盖普通路径、`.`、嵌套路径和 `is_temp` 不变。
- `object_test.*:object.*` 共 20 项通过，结束后堆残留为零。
- 未引入常驻缓存、对象字段或额外分配。
