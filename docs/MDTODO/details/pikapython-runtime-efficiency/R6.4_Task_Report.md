# R6.4 审阅与后续债务

## 边界审阅

- 产品改动仅涉及 `src/PikaObj.c` 的单段 host-object 查询和对应对象单测。
- 未修改 parser、字节码格式、VM 指令、对象布局或 API；未增加常驻 RAM、分配和缓存。
- 路径长度断言位于早退之前，保持超长路径错误合同；嵌套路径和 `is_temp` 语义由单测固定。
- minimal/nano 编译掉快速路径，避免该配置的 32 字节临时 text 增量。

## 踩坑固化

- Debug/gcov 会因新增分支产生 coverage counter，制造伪 `bss/text` 增量；资源证据必须来自 Release 或固件等价构建。
- Linux 的 `pika_config.h` 只有在 `PIKA_CONFIG_ENABLE=ON` 时生效；性能 A/B 还必须核对 Arg cache/PikaMemInfo 输出一致。
- 上述规则已更新到 PikaPython skill，减少后续重复试探。

## 后续债务

- 最终 gprof 主要 self 热点为 VM frame `25.65%`、指令处理 `6.05%`、`obj_setArg_noCopy` `6.05%`；下一轮仍须重新 benchmark 后只选一个局部热点。
- 隔离基线的合法组合条件误报已创建 [#369](https://github.com/pikasTech/PikaPython/issues/369)，留给后续语法正确性批次在完整默认包中复现和修复。
