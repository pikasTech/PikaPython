# R12.3.1 任务报告

## 结果

修复 Python 3 运算符运行语义中的三项差异：`**` 改为右结合，`and`/`or` 改为短路求值，并返回决定结果的原操作数而非强制布尔值。实现保持现有 parser/AST/ASM/VM 架构，不扩大语法范围。

## 实现

- parser 对顶层 `**` 从第一个运算符处分割，使 `2 ** 3 ** 2` 构造为 `2 ** (3 ** 2)`；其他左结合运算符继续使用最后分割策略。
- AST 到 ASM 阶段为 `and`/`or` 生成现有 `JEZ`/`JNZ` 控制流，不再生成会预先求值双方的 `OPT`。
- 左值暂存在按表达式深度区分的内部 `$scN` 局部值，右侧嵌套逻辑表达式的跳转距离按实际 block marker 计算；VM 指令与热分派路径无改动。

## 测试

隔离 Linux Release 树执行 `pull-core.sh` 后重建 `pikascript_test`。最终运行完整 `parser.*`、`except.try_while_return`、既有 `pikaMain.and_or_not`/`a_mm` 及新增运行语义测试，共 19/19 通过。新增覆盖：

- `2 ** 3 ** 2 == 512` 和 `(2 ** 3) ** 2 == 64`。
- `False and`、`True or` 不执行带副作用的右侧；相反分支只执行一次。
- 布尔、整数、字符串操作数返回合同。
- 三项链式表达式、左右嵌套短路及嵌套右侧实际执行。
- 测试结束 `pikaMemNow() == 0`。

## 资源与后续

正确性阶段没有增加 `PikaVMFrame` 字段，也没有修改 VM handler 或主分派热路径。短路 ASM 增加跳转、内部临时值和汇合指令；运行帧存活期间最多按短路嵌套深度保留 `$scN` 局部值。其 default/minimal Flash、峰值 RAM 和 runtime 影响按两阶段原则移交 R12.4 独立量化与回收，不以 parser 性能优先于 runtime 性能。
