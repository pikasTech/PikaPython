# R12.3.2 任务报告

## 结果

修复两项 Python 3 运行语义差异：

- 默认参数在函数定义时求值一次，后续调用复用同一对象；显式实参仍覆盖默认值。
- 函数内存在局部绑定时，赋值前读取不再回退到同名全局，而是输出可见的 `UnboundLocalError`。

实现保持现有 parser/ASM/VM 架构，不增加 `PikaVMFrame` 或 method property 字段，也不扩大 nano/minimal 已裁掉的默认参数语法范围。

## 实现

- default 配置在 `DEF` 前生成默认表达式 ASM，并用参数名把结果送入既有调用栈；因此表达式、可变对象和副作用只在定义时执行。
- `DEF` 一次扫描已生成的函数字节码，收集顶层 `OUT`/`DEL`、嵌套 `DEF`/`CLS` 形成的局部绑定，并排除 `global`、属性、下标和内部变量；嵌套函数/类函数体不会污染外层局部集合。
- 默认值与局部名保存在定义上下文的单个 heap-struct 中，以完整 method 地址隔离不同 bytecode frame；结构随上下文确定析构，不创建隐藏 `PikaObj`，不进入 GC 扫描热路径。
- 调用参数绑定后只做已有 Args 哈希查找与拷贝；缺失局部槽初始化为现有 `ARG_TYPE_UNDEF`。REF 在全局回退前识别该类型并输出 `UnboundLocalError: local variable '<name>' referenced before assignment`。
- nano/minimal 继续沿用既有语法裁剪：默认参数测试由 `!PIKA_NANO_ENABLE` 隔离；局部未绑定错误在 default/minimal 均生效。

## 测试

隔离 Linux Release 树执行 `pull-core.sh` 后重建两档 `pikascript_test`：

- default 定向与回归 28/28 通过：15 个 `parser.*`、既有 default/kwargs、R12.3.1 运算语义、异常控制流及本轮 4 个新增测试。
- minimal 隔离全量 183/183 通过；新增 `UnboundLocalError`、参数读取、赋值后读取、嵌套作用域和 #356 顶层函数引用均通过。
- 新增覆盖定义域标量、可变 list 默认值、定义副作用只执行一次、显式覆盖、class method 默认值、多次 `obj_run` 的 bytecode frame 隔离、`global`、参数和嵌套函数边界。
- 所有新增测试结束均满足 `pikaMemNow() == 0`。

一次未按既有可选组件隔离口径运行的 default 宽泛套件，被 PikaCV/线程/文件资产等前置测试污染全局状态后产生 320 个级联失败；本轮语法定向在独立进程复跑全部通过，该无效尝试不计入验收。

## 资源与后续

当前 core `TOTALS`：

- default：`text=1079326 data=125784 bss=24338`。
- minimal：`text=1028379 data=47180 bss=23074`。

静态 data/bss 相对既有门槛没有变化。动态成本是每个含默认值或局部绑定的 Python 函数持有一个 heap-struct 及对应 Args 项，调用时按该函数局部名数量初始化未绑定槽；没有每次调用扫描字节码，也没有 VM 主分派改动。R12.4 需基于 R12.3 前精确基线分离量化本轮 text、函数定义常驻堆、调用分配次数和 runtime，并在不改变错误合同的前提下回收。
