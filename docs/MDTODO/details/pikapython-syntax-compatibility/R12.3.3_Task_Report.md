# R12.3.3 任务报告

## 结果

修复三组 Python 3 运行语义：

- list comprehension 的简单名称目标不再泄漏、覆盖或删除外层同名变量；原值为 `None` 与原本不存在可正确区分，函数局部作用域同样恢复。
- 支持一个星号目标的 list/tuple 解包，星号目标结果统一为 list。
- 定长解包在写入任何目标前校验数量，过少和过多分别抛出可捕获的 `ValueError`；失败不再部分覆盖旧变量，RHS 只求值一次。

实现只扩展既有 parser sugar 和生成源码，不新增 VM 指令、frame 字段或作用域对象，不改变 nano/minimal 已裁掉的语法范围。

## 实现

- 推导式对简单引用目标生成隐藏旧值和存在标记；用既有 `try/except NameError` 区分“不存在”与值为 `None`，循环结束后恢复或删除目标并清理隐藏变量。
- 解包先把 RHS 保存到 `$tmp`，用 `len` 完成全部数量校验，再按前缀、星号 slice 和后缀索引写目标。
- 多星号目标继续进入既有 `SyntaxError` 路径；本轮不扩展任意 iterator、嵌套或括号/list target。
- full-only 代码使用 `PIKA_NANO_ENABLE` 编译门禁，minimal 保留原推导式构造和 multi-assign 早退路径。

## 测试

- 新增 3 个 GTest，覆盖顶层已有/缺失/`None` 目标、函数局部目标、list 与 tuple 星号解包、固定与星号解包过少、固定解包过多、失败原子性及 RHS 单次求值。
- default 定向 3/3 通过；全部 `parser.*` 与 `pikaMain.python3_*` 23/23 通过；相关循环 4/4 通过。
- minimal 排除 LVGL、网络及其他可选组件后全量 183/183 通过。
- 所有新增运行测试结束满足 `pikaMemNow() == 0`。
- default 宽泛套件再次因既有可选组件/全局状态污染产生 323 个级联失败，与 R12.3.2 已记录的无效口径一致，不计入验收。
