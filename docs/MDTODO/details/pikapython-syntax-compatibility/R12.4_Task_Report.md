# R12.4 任务报告

## 结果

在不改变 R12.3 语义和错误合同的前提下完成资源、生成代码与 runtime 复核：

- 固定解包成功路径由两次 `<`/`>` 校验改为一次 `!=`；只有错误路径再执行 `<` 分类。
- 隐藏长度变量由 `$unpack_len` 缩短为 `$ul`，减少每个生成字节码常量池。
- nano/minimal 对推导式隔离和 multi-assign 使用完整预处理门禁，保留原单次字符串构造路径。

## 验收

- default 定向语义 3/3 通过；`parser.*:pikaMain.python3_*` 23/23 通过。
- minimal 在 R12.3.3 最终版本隔离全量 183/183 通过；R12.4 变更由 full-only 门禁排除，最终 core 重建尺寸不变。
- default core：`text 1080717 -> 1080693`，回收 24 B；`data=125784`、`bss=24338` 不变。
- minimal core：`text=1028379 data=47180 bss=23074`，相对 R12.3 前零增量。
- 三项定向测试的总分配次数由 132846 降至 132788，减少 58 次；语义测试结束堆占用仍为 0。
- 高主机负载下，当前与旧 baseline 的 `function_call_1000` CPU 中位数为 7.27 ms/7.34 ms；`while_loop_10000` 为 5.87 ms/5.66 ms。后者约 3.7% 波动，且本轮无 VM 改动，不作为性能收益或退化结论。

## 剩余差额

相对 R12.3.2，default 仍增加 1367 B text，来自 full-only 推导式作用域保存和解包数量/星号处理本体；静态 RAM 与 minimal 均无增量。继续回收需要把语义逻辑转移到 VM 指令或 runtime 热路径，违反本轮“不改架构且 runtime 优先”的约束，因此保留并明确记录，不回退正确性。
