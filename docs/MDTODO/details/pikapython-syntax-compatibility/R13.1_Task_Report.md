# R13.1 任务报告

## 探测结果

在现有 operator、函数调用、比较和异常 parser 路径中确认四组问题：

1. 整数 `1 // 0` 与 `1 % 0` 均直接触发 SIGFPE，独立进程退出码为 136；`-3 // 2`、`-3 % 2`、`3 // -2`、`3 % -2` 分别错误返回 `-1/-1/-1/1`，CPython 3 预期为 `-2/1/-2/-1`。
2. `def f(a, b=5)` 后调用 `f(1, a=2)` 未抛出 `TypeError`，而是静默执行并得到错误结果；合法邻居 `f(1, b=2)` 正常。
3. `3 < middle() < missing_name` 不按 Python 比较链短路，继续读取 `missing_name` 并产生 `NameError`。比较链不在项目声明的语法子集内，本轮应稳定拒绝，保留单个比较和括号嵌套比较。
4. README 与 `assets/syntax.mdpp` 明确标记 `finally` 不支持，但 parser 在 `try`/`except` 后接受该 block，运行时静默跳过 block，包括正常路径和 return 路径；本轮应在解析期稳定拒绝。

链式赋值 `a = b = expr` 当前已返回语法错误，支持它会扩大语法范围，因此排除。后续修复不新增 VM 指令或持久状态，并分别覆盖崩溃、错误结果、错误绑定和错误接受。
