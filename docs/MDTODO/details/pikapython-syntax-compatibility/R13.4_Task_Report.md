# R13.4 任务报告

## 资源结果

相对 R12 最终基线：

- default：`text 1080693 -> 1081861`，增加 1168 B；`data=125784`、`bss=24338` 不变。
- minimal：`text 1028379 -> 1029539`，增加 1160 B；`data=47180`、`bss=23074` 不变。
- `FunctionArgsInfo` 仍为 88 B；新增关键字计数字段复用原对齐空间，没有栈帧尺寸增量。
- 重复参数检测复用可写 typelist 和既有关键字字典，不增加堆分配。

## Runtime 复核

- 无关键字实参时增加快速返回，声明 `**kwargs` 的普通位置调用不再扫描参数名；该 runtime 优化增加 64 B text，按“runtime 优先于 parser/Flash 微优化”原则保留。
- 同机 A/B 基准在高负载下，启用检测的 `function_call_1000` CPU 中位数为 7.10 ms，临时禁用检测为 7.63 ms；`while_loop_10000` 为 6.10/6.28 ms。噪声大于差异，没有可信的 runtime 退化。
- 将快速门禁移到调用点会再增加 128 B text，已回退；未为减小 Flash 把检查转移到 VM 主分派或引入常驻元数据。

保留的约 1.1 KiB text 是除零保护、Python 符号语义、明确错误诊断和 parser 拒绝逻辑本体。继续回收需要单独做编译器级代码布局或共享错误路径优化，不能回退本轮错误合同。
