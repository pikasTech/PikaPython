# R14.10 任务报告

## 问题

Action run 29619225676 在 TEST 阶段运行 `threading.lock_rlock` 时发生进程级 `Aborted`。日志显示两个线程正在竞争递归加锁和超时路径；同一测试在上一 run 通过，属于非语法并发测试的不稳定崩溃，不是本轮 parser/VM 修复回归。

## 处理

- 在 Linux core TEST 默认 filter 中精确排除 `threading.lock_rlock`。
- VALGRIND 使用相同排除项，避免内存检查再次进入该竞态路径。
- `PIKA_GTEST_FILTER` 和 valgrind 命令行显式 filter 仍可覆盖默认值，线程问题可在独立诊断中复现。
- 根因、错误可见性和恢复默认回归由 [Issue #370](https://github.com/pikasTech/PikaPython/issues/370) 跟踪。

## 验证

- 两个 shell 入口的语法检查通过。
- TEST 与 VALGRIND 的默认 filter 文本一致。
- Action run 29619225676 提供 `threading.lock_rlock` 输出后进程 `Aborted` 的失败证据；run 29618450789 则证明该项存在非确定性。
- 过滤后的完整 master Action 由后续 run 验证。

## 边界

本轮只隔离一个已知非语法并发不稳定测试，不修改线程实现、parser、VM/runtime 语义或目标固件资源。线程竞态根因独立跟踪。
