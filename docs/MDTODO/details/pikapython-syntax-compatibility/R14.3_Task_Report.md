# R14.3 任务报告

## VM 热路径优化

- 将参数 typelist 的逗号、星号和等号三次完整扫描合并为一次字符遍历。
- 直接在既有可写 typelist 中从尾部剥离 *args 与 **kwargs，删除第二个 METHOD_TYPE_LIST_MAX_LEN 临时缓冲、strCopy 和重复反向扫描。
- 每次默认参数或变长参数调用减少 256 B 临时堆申请，未增加对象字段、VM 指令或常驻缓存。
- kwargs 名称目前只在 Arg 中保留 hash，真实名称依赖常量池恢复；继续消除该扫描需要新增元数据或缓存，本轮按架构与 RAM 边界不实施。

## A/B 判定

- 两组交错 A/B 的 CPU CV 约为 4% 至 18%，覆盖多数差异。
- 默认参数与 starred 路径方向为改善，但噪声不足以宣称稳定加速。
- 结论限定为：确定减少扫描和 256 B 临时堆，未发现可信 runtime 退化。
