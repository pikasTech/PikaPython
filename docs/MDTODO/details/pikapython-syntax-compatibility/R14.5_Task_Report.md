# R14.5 任务报告

## 实现

- kwargs 收集阶段只保存现有 keyword hash 和数字 fallback，不再为每个 keyword 重复扫描 bytecode constant pool。
- 参数绑定完成后仅对仍需进入 `**kwargs` 的键一次遍历 constant pool，找到全部名称后提前结束。
- 短名称原位复用数字 fallback 的 `Arg` 容量；长名称才走既有容器替换路径，不增加 frame 字段、常驻缓存或静态 RAM。
- 保持 kwargs 顺序、首个 hash 匹配名称和找不到名称时的数字 hash fallback，不扩大语法范围。

## 测试

- 新增长 keyword 回归，覆盖占位容量不足时的替换分支和 kwargs 顺序。
- default 既定语法/runtime 门禁：36/36 通过。
- kwargs、默认参数、必选参数、位置/关键字冲突和 native method 定向：5/5 通过。
- minimal 隔离全量：225/225 通过。
- 网络、LVGL 和非语法可选组件不作为本轮门禁；minimal 全量中的缺包输出为既有预期行为。

## 性能

- 新增 `function_call_kwargs_large_constpool_1000`，以 20 个无关字符串常量放大名称恢复开销，并保留 while、默认参数、普通 kwargs 和 starred 控制基准。
- 同机交错 A/B 时主机 load average 约 38-50，CPU CV 约 3.5%-11.5%。
- 第二组 10 次 CPU 均值：while 基线/优化 6.57/6.93 ms，普通 kwargs 5.16/5.40 ms，大常量池 kwargs 5.85/5.71 ms。
- 变化均未稳定超过噪声，只能判定“无可信退化”；不宣称稳定 runtime 收益。实现消除了随 keyword 数量重复计算 constant hash 的渐进债务。

## 资源

- default：text 1082053、data 125784、bss 24338；相对 R14 text 增加 192 B，data/bss 不变。
- minimal：text 1029603、data 47180、bss 23074；相对 R14 text 增加 128 B，data/bss 不变。
- 未增加静态 RAM、frame 字段或持久缓存；有界 Flash 增量用于一次扫描和长名称安全替换。
- 隔离源码配置已恢复 default，`git diff --check` 通过。
