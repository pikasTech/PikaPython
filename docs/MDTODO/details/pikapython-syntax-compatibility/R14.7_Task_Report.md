# R14.7 任务报告

## 结论

链式比较门禁在整条表达式上累计比较运算符，导致 `a < b and b < c` 与 `a < b or c < d` 被误判为不支持的链式比较。

## 修复

- 先确定顶层运算符，再执行链式比较数量检查。
- 顶层为 `and` 或 `or` 时先按现有 AST 递归分组；每个分支仍独立执行链式比较门禁。
- 恢复已有逻辑比较子集，不新增表达式能力；`1 < 2 < 3` 和混合链式比较仍稳定返回语法错误。
- 只调整 parser 现有检查顺序，不新增持久状态、VM 指令、RAM 字段或 runtime 热路径工作。

## 验证

- 逻辑比较、非法链式比较、短路语义及两个原失败集成测试：5/5 通过。
- 隔离 LVGL 和网络后的 Linux default 全量：467/467 通过，耗时 26.194 秒。
- 构建使用宿主 mbedTLS 3 的临时 API 别名；该别名只存在于本地 CMake 配置，没有进入仓库变更。
