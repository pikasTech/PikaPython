# R14.8 任务报告

## 问题

Linux Action 的 core TEST 已默认隔离网络测试并带超时，但 VALGRIND 仍无参数执行全部 GTest，也没有进程时限。网络类用例可能等待外部条件，且异常输出会持续占用日志，导致内存检查阶段长期不终止。

## 修复

- VALGRIND 默认复用 core TEST 的网络过滤合同：排除 mqtt、network、socket、requests 和阻塞 JRPC 用例。
- 保留命令行 `--gtest_filter` 的显式覆盖，并支持 `PIKA_GTEST_FILTER` 环境变量覆盖默认值。
- 使用十分钟硬超时约束 valgrind 子进程。
- 启用 shell 失败传播，构建、valgrind 错误或超时均使 Action 明确失败。

## 验证

- `sh -n port/linux/valgrind.sh` 通过。
- shell 跟踪确认显式 filter 正确展开为单一 `--gtest_filter=...` 参数。
- 同一提交前的 Linux core TEST 已完成 467/467；旧 VALGRIND 入口则停留在运行中，证明该边界缺失确实影响流水线收敛。
- 当前宿主未安装 valgrind，完整内存检查由 Ubuntu 22 GitHub Action 验证。

## 边界

本轮只调整 Linux CI 测试入口，不改变 parser、VM/runtime 语义、架构或目标固件 RAM/Flash。
