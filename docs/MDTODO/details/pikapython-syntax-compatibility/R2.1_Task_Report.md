# R2.1 任务报告

- 已复现 [#349](https://github.com/pikasTech/PikaPython/issues/349)：`func(a=1)` 在同时声明必选参数与 `**kwargs` 时曾进入错误绑定路径。
- 已复现 [#366](https://github.com/pikasTech/PikaPython/issues/366)：声明超过 `PIKA_ARG_NUM_MAX` 的函数会在固定 `argv` 暂存区写入前缺少槽位上限检查；`stack_pushSize()` 也曾先写元数据再检查容量。
- 根因分别属于关键字参数绑定边界、固定参数槽位边界和错误可见性，不需要扩大语法或修改 VM 架构。
