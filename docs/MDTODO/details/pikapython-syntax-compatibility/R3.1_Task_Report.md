# R3.1 任务报告

## 复现结论

- 合法变长参数：Python 函数通过 `*args` 接收 25 个位置实参，执行通过，不应受固定参数槽位上限误伤。
- 固定槽位溢出：原生方法声明 17 个固定参数时超过 `PIKA_ARG_NUM_MAX`，现返回可捕获的 `OverflowError: arg limit`，不再越界写或死机。
- 非法语法：缺少冒号、缺少右括号、残缺函数声明、函数参数开头逗号和连续逗号均应在编译阶段返回 `PIKA_RES_ERR_SYNTAX_ERROR`。
- 合法边界：空参数列表、单个尾逗号、默认值括号或字符串内部逗号必须继续接受。

## 根因

- 固定参数死机来自 VM 参数槽位在边界检查前写入，且函数声明的原始槽位数未保留。
- 部分非法 AST/ASM 生成失败未向 `pika_lines2Bytes()` 传播。
- 函数参数连续逗号会被默认参数重写流程接受，需要在同一次顶层 token 遍历中识别空参数项。
- Linux 构建由 `pull-core.sh` 生成 core 副本；定向复测必须同步源文件后再构建，避免验证旧副本。

## 定向证据

NC01 隔离构建运行 7 项定向测试，结果 7/7 通过，覆盖合法 25 实参、固定 17 槽位可捕获错误、栈元数据边界、多类非法语法、合法逗号边界及既有嵌套解包拒绝。
